# Exercice : Couverture de Code avec PHPUnit

Durée estimée : 3 heures
(Incluant : installation des outils, réalisation des tests, génération des rapports et présentation.)

Chaque groupe (ou étudiant, selon l’effectif) devra analyser différents outils de génération de couverture de code, puis présenter ses conclusions.

La présentation devra aborder :
- la facilité ou difficulté d’installation,
- la simplicité d’utilisation,
- les avantages,
- les limites ou inconvénients de chaque outil.

## Travail à réaliser
### 1. Installer et utiliser un moteur de couverture

En se basant sur l'exercice précédent "Tests fonctionnels BDD", choisir et configurer l’une des deux extensions compatibles PHPUnit :
- PCOV
- Xdebug

L’extension installée doit permettre de collecter les données de couverture lors de l’exécution des tests PHPUnit.

### 2. Générer plusieurs formats de rapports de couverture

À partir de la même suite de tests, produire différents rapports parmi :

- Clover
- Cobertura
- Crap4J
- PHPUnit (format XML natif)

Chaque format doit être généré, analysé et comparé.


## Objectif pédagogique

L’objectif final est de disposer d’un panorama complet des outils disponibles pour la couverture de code avec PHPUnit, et de comprendre :

- leurs spécificités,
- leurs cas d’usage,
- les différences entre les formats,
- ce qu’ils apportent en termes de qualité logicielle.

Chaque groupe devra faire une courte démonstration ou synthèse pour illustrer son travail.
